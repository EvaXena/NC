## 1. 项目结构

```
/root/nautilus/NAUTILUS/
├── dataset/                          # 数据集
│   ├── train/                        # 训练图片 (2000 张)
│   ├── test/                         # 测试图片 (499 张)
│   ├── train_nautilus.json           # 训练标注
│   ├── test_nautilus.json            # 测试标注
│   └── convert_to_nautilus_format.py # 数据格式转换脚本
├── qwen-vl-finetune/                 # 模型训练与推理代码
│   ├── qwenvl/                       # 模型核心代码
│   │   ├── nautilus_model/           # Nautilus 模型定义
│   │   ├── train/                    # 训练逻辑 (trainer, train_qwen.py)
│   │   └── data/                     # 数据集配置 (__init__.py)
│   └── scripts/                      # 脚本目录
│       ├── nautilus_finetune/
│       │   └── coral_det_lora.sh     # ★ 训练启动脚本

│       ├── batch_inference.py        # ★ 批量推理脚本

│       ├── visualize_results.py      # ★ 双栏可视化（GT vs Nautilus）

└── 使用说明.md                        # 本文件
```

## 2. 预训练权重位置

| 权重 | 路径 | 说明 |
|------|------|------|
| 基座模型 | `Qwen/Qwen2.5-VL-7B-Instruct` | HuggingFace 自动下载，缓存在 `~/.cache/huggingface/` |
| DINOv2 编码器 | `/root/autodl-tmp/nautilus_checkpoints/weight/dino_vitl.pth` | Nautilus 双编码器的视觉分支 |

## 3. 训练好的 Checkpoint

存放在 `/root/autodl-tmp/nautilus_checkpoints/`：

| Checkpoint | 大小 | 说明 |
|------------|------|------|
| `checkpoint-500` | ~22G | 第 500 步（含完整优化器状态） |
| `checkpoint-1500` | ~15G | 第 1500 步 / 最终（优化器状态较小，因 cosine scheduler 末期动量/方差衰减） |

## 4. 完整训练流程

### 4.1 训练（LoRA 微调）

```bash
cd /root/nautilus/NAUTILUS/qwen-vl-finetune
bash scripts/nautilus_finetune/coral_det_lora.sh
```

**主要参数**（在 `coral_det_lora.sh` 中修改）：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `pretrain_llm` | `Qwen/Qwen2.5-VL-7B-Instruct` | 基座模型 |
| `dino_vitl_weight` | `/dev/shm/nautilus_train/weight/dino_vitl.pth` | DINOv2 权重路径 |
| `lr` | `2e-5` | 主学习率 |
| `nautilus_lr` | `2e-7` | Nautilus 模块学习率 |
| `batch_size` | `4` | 每卡 batch size |
| `num_train_epochs` | `3` | 训练轮数 |
| `lora_rank` | `128` | LoRA 秩 |
| `lora_alpha` | `256` | LoRA alpha |
| `output_dir` | `/dev/shm/nautilus_train/output/nautilus_coral_det_lora` | checkpoint 输出目录 |
| `save_steps` | `500` | 每 N 步保存一次 |

**注意**：默认输出到 `/dev/shm`（内存盘，重启后丢失）。训练完成后务必将 checkpoint 备份到持久存储：
```bash
cp -r /dev/shm/nautilus_train/output/nautilus_coral_det_lora/checkpoint-* /root/autodl-tmp/nautilus_checkpoints/
```

### 4.2 合并 LoRA 权重

将 LoRA adapter 合并回基座模型，生成可直接加载的完整模型：

```bash
cd /root/nautilus/NAUTILUS/qwen-vl-finetune

python scripts/merge_lora.py \
    --input_path /root/autodl-tmp/nautilus_checkpoints/checkpoint-1500 \
    --output_path /dev/shm/nautilus_coral_det_merged \
    --base_model Qwen/Qwen2.5-VL-7B-Instruct \
    --dino_path /root/autodl-tmp/nautilus_checkpoints/weight/dino_vitl.pth
```

| 参数 | 说明 |
|------|------|
| `--input_path` | LoRA checkpoint 目录 |
| `--output_path` | 合并后模型保存路径（约 17G） |
| `--base_model` | 基座模型名称或路径 |
| `--dino_path` | DINOv2 权重路径 |

**注意**：合并后需要补充 processor 配置文件，否则推理会报错：
```python
from transformers import AutoProcessor
processor = AutoProcessor.from_pretrained("Qwen/Qwen2.5-VL-7B-Instruct")
processor.save_pretrained("/dev/shm/nautilus_coral_det_merged")
```

### 4.3 批量推理（Nautilus 微调模型）

```bash
cd /root/nautilus/NAUTILUS/qwen-vl-finetune

python scripts/batch_inference.py \
    --checkpoint /dev/shm/nautilus_coral_det_merged \
    --test_json /root/nautilus/NAUTILUS/dataset/test_nautilus.json \
    --image_dir /root/nautilus/NAUTILUS/dataset/test \
    --output /root/autodl-tmp/nautilus_eval_results.json \
    --max_samples 20
```

| 参数 | 说明 |
|------|------|
| `--checkpoint` | 合并后的模型路径 |
| `--test_json` | 测试集标注文件 |
| `--image_dir` | 测试图片目录 |
| `--output` | 推理结果 JSON 输出路径 |
| `--max_samples` | 最多推理多少张（0 = 全部 499 张） |

### 4.4 一键合并+推理

```bash
bash /root/nautilus/NAUTILUS/run_eval.sh
```

该脚本自动执行：合并 LoRA → 批量推理全部 499 张测试图。路径配置在脚本开头修改。

### 4.5 批量推理（原始 Baseline 模型）

使用未微调的原始 Qwen2.5-VL-7B-Instruct 对相同测试集推理，用于与 Nautilus 微调模型做对比：

**脚本位置**：`qwen-vl-finetune/scripts/batch_inference_baseline.py`

```bash
cd /root/nautilus/NAUTILUS/qwen-vl-finetune

python scripts/batch_inference_baseline.py \
    --checkpoint Qwen/Qwen2.5-VL-7B-Instruct \
    --test_json /root/nautilus/NAUTILUS/dataset/test_nautilus.json \
    --image_dir /root/nautilus/NAUTILUS/dataset/test \
    --output /root/autodl-tmp/nautilus_eval_results_baseline_20.json \
    --max_samples 20
```

| 参数 | 说明 |
|------|------|
| `--checkpoint` | 原始模型名称或路径（无需合并，直接加载） |
| `--test_json` | 测试集标注文件 |
| `--image_dir` | 测试图片目录 |
| `--output` | 推理结果 JSON 输出路径 |
| `--max_samples` | 最多推理多少张（0 = 全部 499 张） |

> **与 Nautilus 推理脚本的区别**：原始模型使用 transformers 自带的 `Qwen2_5_VLForConditionalGeneration`，不需要 DINOv2 编码器，也不需要 `double_image_tokens`（Nautilus 双编码器专用）。

### 4.6 可视化 — 双栏对比（GT vs Nautilus）

将 Ground Truth 和 Nautilus 微调模型的预测 bbox 左右并排展示：

**脚本位置**：`qwen-vl-finetune/scripts/visualize_results.py`

```bash
cd /root/nautilus/NAUTILUS/qwen-vl-finetune

python scripts/visualize_results.py \
    --results_json /root/autodl-tmp/nautilus_eval_results_20.json \
    --image_dir /root/nautilus/NAUTILUS/dataset/test \
    --output_dir /root/autodl-tmp/nautilus_vis
```

| 参数 | 说明 |
|------|------|
| `--results_json` | 推理结果 JSON（`batch_inference.py` 的输出） |
| `--image_dir` | 测试图片目录 |
| `--output_dir` | 可视化图片输出目录 |

**输出效果**：每张图生成一张并排对比图
- 左侧：**绿色框** — Ground Truth
- 右侧：**红色框** — Nautilus Prediction
- 标题栏显示各自检测的目标数量

### 4.7 可视化 — 三栏对比（GT vs Nautilus vs Baseline）

将 Ground Truth、Nautilus 微调模型、原始 Baseline 模型的预测 bbox 三栏并排展示，直观对比微调效果：

**脚本位置**：`qwen-vl-finetune/scripts/visualize_compare.py`

```bash
cd /root/nautilus/NAUTILUS/qwen-vl-finetune

python scripts/visualize_compare.py \
    --nautilus_json /root/autodl-tmp/nautilus_eval_results_20.json \
    --baseline_json /root/autodl-tmp/nautilus_eval_results_baseline_20.json \
    --image_dir /root/nautilus/NAUTILUS/dataset/test \
    --output_dir /root/autodl-tmp/nautilus_vis_compare
```

| 参数 | 说明 |
|------|------|
| `--nautilus_json` | Nautilus 微调模型的推理结果 JSON |
| `--baseline_json` | 原始 Baseline 模型的推理结果 JSON |
| `--image_dir` | 测试图片目录 |
| `--output_dir` | 可视化图片输出目录 |

**输出效果**：每张图生成一张三栏对比图
- 左侧：**绿色框** — Ground Truth
- 中间：**橙色框** — Nautilus 微调模型 Prediction
- 右侧：**红色框** — 原始 Baseline Prediction
- 标题栏显示各模型检测的目标数量

## 5. 环境要求

- Conda 环境：`nautilus_qwen`
- 激活命令：`conda activate nautilus_qwen`
- GPU：需要至少一张支持 flash_attention_2 的 GPU（推理约需 17G 显存）
- 训练推荐：单卡 80G（A100/A800）或多卡
